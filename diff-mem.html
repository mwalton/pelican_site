Neural Stacks & "Learning to Transduce with Unbounded Memory"

Posted on Wed 15 August 2018 in RNNs

Earlier this year I became interested in results from DeepMind demonstrating various forms of 'differentiable memory' (Neural Turing Machines etc.) and the idea of attention-based memory modules as in important component to reinforcement learning in partially observable & sparse reward environments (eg MERLIN). However these models were waaay to big and complicated for me to reproduce as a pet project; I found there was some older work proposing a more approachable model implementing a neural-flavored variation a much simpler datastructure (a stack). These are notes on a partial reproduction of results from Learning to Transduce with Unbounded Memory to teach a simple network to reverse a sequence of bits.

class Stack:
    def __init__(self):
        self.s = []
        
    def push(self, v):
        self.s.append(v)
    
    def pop(self):
        v = self.s[-1]
        self.s = self.s[:-1]
        return v

ok cool, 😪... on to the fun stuff¶

We going to make a stack-like datastructure with continuous push & pop operations (intuitively corresponding to degrees of certainty for each operation) composed of functions that are almost everywhere differentiable (so we can backpropigate gradients through the stack module)

Update rules for the stack are defined in the original paper by the following 3 equations:

Where:

  • $V_t$ is the contents of the stack, a list / matrix / tensor (your implementation may vary) of row vectors $[\boldsymbol{v}_i]_{i=1}^t$ where on each timestep $t$ a new $\boldsymbol{v}_t$ is pushed onto the stack
  • $s_t$ is the current weight on each row of $V_t$ at time $t$
  • $u_t,d_t$ are pop and push strengths respectively
  • $r_t$ is the return value of the stack update operation

The original paper and the tutorial above provide more in-depth info motivating these design choices and intuitions for how these parameters behave.

class NeuralStackNp:
    def __init__(self):
        self.V = list()
        self.s = list()
    
    def step(self, v, u, d):
        ''' step function both pushes and pops; maybe unintuitive at first
            but setting d and/or u to 0 negates push / pop respectively
            
            v : new value to push
            d : push strength (setting to 0 means no push)
            u : pop strength 
        '''
        # update V
        self.V.append(v)
        
        # update s
        for i in range(len(self.s)):
            self.s[i] = max(0, self.s[i] - max(0, u - sum(self.s[i+1:])))
        
        self.s.append(d)
        
        # calculate the return value
        r = np.sum([min(s, max(0, 1 - sum(self.s[i+1:]))) * self.V[i] for i, s in enumerate(self.s)], axis=0)
        
        return r

class NeuralStack(torch.nn.Module):
    def __init__(self):
        super(NeuralStack, self).__init__()
        self.V = Variable(torch.FloatTensor(0))
        self.s = Variable(torch.FloatTensor(0))

    def forward(self, v, u, d):
        ''' step function both pushes and pops; maybe unintuitive at first
            but setting d and/or u to 0 negates push / pop respectively

            @param v : [batch_size, mem_width] new value to push
            @param u : [batch_size,] pop strength
            @param d : [batch_size,] push strength (setting to 0 means no push)

            self.V in [T, batch_size, mem_width]
            self.s in [T, batch_size]

            returns r in [batch_size, mem_width]
        '''
        # update V
        self.V = torch.cat([torch.unsqueeze(v, dim=0), self.V])

        # update s
        if self.s.shape[0] > 0:
            inner_sum = torch.cumsum(self.s, dim=0)
            offset_sum = torch.cat([torch.zeros(1, *inner_sum.shape[1:]), inner_sum])[:-1]
            self.s = F.relu(self.s - F.relu(u - offset_sum))

        # append s w/ push weiht
        self.s = torch.cat([torch.unsqueeze(d, dim=0), self.s])

        # calculate the return value
        inner = torch.cumsum(torch.cat([torch.zeros_like(torch.unsqueeze(self.s[0], dim=0)), self.s]), dim=0)[:-1]
        weights = torch.min(self.s, F.relu(1.0 - inner))
        r = torch.sum(self.V * torch.unsqueeze(weights, dim=-1), dim=0)

        return r

    def reset(self):
        self.V = Variable(torch.FloatTensor(0))
        self.s = Variable(torch.FloatTensor(0))

Neural Pushdown Automata¶

Although flexible modeling tools for many sequential prediction problems, recurrent neural networks (modern variants such as LSTM included) struggled for years to solve tasks that are straightfordly accomplished with vastly simpler hand-coded solutions; for example: the simple stack in the first cell reverses a sequence, something RNNs surprisingly struggle to accomplish! In learning to transduce, it is observed that neural pushdown automata vastly outperform conventional recurrent models on such tasks.

def eval(y_pred, y_true, loss_fn):
    idx = y_true != 2
    y_pred_ = y_pred[idx, :]
    y_true_ = y_true[idx]
    if len(y_pred_) == 0:
        return None, None

    _, y_pred_max = torch.max(y_pred_, 1)

    acc = len(torch.nonzero((y_pred_max == y_true_).data)) / len(y_pred_)
    loss = loss_fn(y_pred_, y_true_)

    return loss, acc

# test the stack controller
n_epoch = 100
batch_size = 16
max_len = 12

train_x, train_y = get_tensors(800, mean_length=10, std_length=2, min_length=1, max_length=max_len)
test_x, test_y = get_tensors(100, mean_length=10, std_length=2, min_length=1, max_length=max_len)

word_size = train_x.shape[-1]

stack = NeuralStack()
controller = SimpleController(input_size=word_size,
                              output_size=word_size,
                              batch_size=batch_size,
                              mem_struct=stack,
                              mem_width=word_size)

loss_fn = torch.nn.NLLLoss()
optimizer = torch.optim.Adam(controller.parameters(), lr=.01, weight_decay=.01)

losses = []
accuracy = []
grads = []

for epoch in tqdm(range(n_epoch)):
    epoch_acc = []
    for i in range(0, len(train_x.data) - batch_size + 1, batch_size):
        controller.reset()
        batch_loss = 0.0
        batch_acc = []

        for j in range(train_x.shape[1]):
            y_pred = controller(train_x[i:i+batch_size, j, :])
            loss, acc = eval(y_pred, train_y[i:i+batch_size, j], loss_fn)
            if loss is None: continue

            batch_loss += loss

            # calculate accuracy
            batch_acc.append(acc)

        losses.append(batch_loss / batch_size)
        epoch_acc.append(np.mean(batch_acc))

        # clear grads from last timestep
        optimizer.zero_grad()

        # compute grads for all learnable parameters of the model
        batch_loss.backward()
        
        # clip the gradients
        #torch.nn.utils.clip_grad_norm_(controller.parameters(), 0.2)
        grads.append(np.concatenate([p.grad.data.numpy().reshape(-1) for p in controller.parameters()]))

        # apply gradient update
        optimizer.step()
    accuracy.append(np.mean(epoch_acc))
grads = np.stack(grads)

It works! ... kinda; in all I found learning to be particularly unstable & sensitive to hyperparameters and am guessing the authors used some additional tricks (fine tuned gradient clipping, regularizations, more extensive hyperparameter search etc.) not mentioned in the original paper. I noticed that gradient would consistently blow up at the beginning of training and the network would converge to one of two local optima: 1. do the thing we want (reproduce the reversed sequence) or 2. predict only 1 or 0 forever; effectively accepting it can do no better than random chance.
